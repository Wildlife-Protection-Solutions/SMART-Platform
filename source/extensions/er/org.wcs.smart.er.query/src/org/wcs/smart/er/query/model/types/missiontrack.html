Survey Mission Track Query

Display all mission tracks that match the given filters. Track type, sampling unit, mission and area filters are applied directly to the track. Observation filters are applied to observations associated with the mission associated with the track.

Mission Track Query vs Mission Query

  • Mission Track Queries generate on one record per track (and include track properties). Mission queries create one record per mission.

  • Mission Track Queries filters are applied to the mission tracks, not the observations. Therefore if you filter on a sampling unit, it will return tracks that match the sampling unit, not observations. Mission queries apply the filters to the observations.

Example Mission Track Query

Results will include one record per mission that matches the given filters. No observation or incident information is included in the results. Mission details are displayed in a tabular window. Mission tracks are displayed in a map window.
