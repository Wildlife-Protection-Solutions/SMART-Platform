SMART-Cybertracker Plug-In

Installing the SMART-Cybertracker Plug-in

Software Requirements:

You must have the Cybertracker Desktop application installed on your computer for the plug-in to function correctly, Cybertracker requires at least version 3.319.

To install the SMART-Cybertracker Plug-in simply:

1) Unzip the Cybertracker zip file after downloading it from the SMART website.

2) Copy the "plugins" folder that from that zip file and paste it into your SMART install directory(where your smart.exe file is) so that the two "plugins" are merged.

3) When you run smart the next time, you will see a "Cyber Tracker" menu to confirm plug-in is installed correctly.

Properties

Selecting the "Cybertracker" -> "Properties" menu option will show you a number of properties that can be configured to modify the behavior of the exported mobile application. The "Application Name" is what the application will be called on the mobile device and it should be noted that exporting an application with the same name more than once will overwrite any existing CT data collection applications already on the device with the same name.

Hover your mouse over any property name or field to get a more detailed description of how it will affect the output application.

Patrol Screens Configuration

Selecting the "Cybertracker" -> "Patrol Screens Configuration" menu option will allow you to customize the metadata collections screens that are shown on the mobile application at the beggining of each Patrol. The 11 meta-data collection screens are shown at the left, select which one you would like to edit and find the relevant details on the right-hand side.

Un-check "Page Visible" to disable the page from showing to the data collection users for any pages you wish to exclude. You can also select a value to set that option to for all patrols if you wish. For example on "Patrol Type" you can un-check "Page Visible" and then select "Ground". Once that application is exported, data is collected and uploaded back to the database, all the patrols from that device will have the patrol type set to "Ground".

Exporting the data Collection Model to a Mobile Device

To export your data collection application to a mobile device:

  1. Select the "Cybertracker" -> "Export ..." button.
  2. Select one of your custom configured data models or "<original data model>" to choose which data model you wish your data collection team to use for this application.
  3. Press "Export".
  4. If this is the first time you are exporting to a particular device you may get on-screen instructions of how to confirm the application install on your mobile device (Android requires this), follow these instructions then return to step #1 and repeat the process.
  5. Press "OK" on the "Export Completed Successfully" dialoge box.
  6. Unplug your device from your computer and test that you see a new CT data collection application with the "Application Name" you selected on the properties page (the default name is "SMART").

Importing Data After Data Collection is Complete

To import data back into your database from a mobile device, plug the device into your computer and then do the following:

  1. Select the "Cybertracker" -> "Import ..." button.
  2. Press the "Import From Device" button.
  3. The system will load all patrols from the mobile device into a temporary table on the screen.
  4. Select one or more (use ctrl or shift to select multiple lines at once) patrols and hit the "Add As New Patrol" button.
  5. <optional>Should you wish to combine patrols with existing ones, use the "Add As New Leg" button and select the patrol you wish to combine them with. This is not usually necessary unless you team split into multiple groups with more than a single mobile collection device.
