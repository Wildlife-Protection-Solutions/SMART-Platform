Tricks and Tips

Shift Select Spacebar

In many parts of the SMART application when many objects (attributes, waypoints, etc...) are toggled on or off the user can toggle them individually, through the Select All/De-select All or through another option.

Using the Shift button and selecting a block of entries and hitting the Space Bar the selected entries will toggle on or off.

FAQ

Q. Adding a gracticule to the map view it often fails to draw, resulting in a red X on the graticule layer in the TOC. How do I get the Gradicule to display?

A. Graticule only support CRS with unit “meter” on all axis.

To specify the projection for the Graticule:

1. Open the Graticule style layer (right click on the layer and pick 'Change Styles' or select the layer and use the edit style button);

2. Pick "Projection" page from the list on the left

3. Select a projection with "meter" on all axis.

4. Apply.

Q. Why is the style editor theme function not working in reports?

A. This is a fundamental problem with the way themeing styler works. It builds themes based on the data in the layer. However when setting up the report there is no data in the layer hence the error. The query data is not generated until the report is run and dates are provided.

The work around; if you want themed styles in your report is to:

1. Run the query in the query perspective and setup the themeing style using this the style editor associated with the query layer.

2. When the themeing style is configured the way you want it, open the style dialog for the layer, select the 'XML' page from the list on the left.

3. Copy the entire XML snippet shown.

4. Switch back to editing your report.

5. Change the style of the map layer in the report which you want to apply the themeing to and paste the copied XML into the "XML" page of this style dialog.

6. Save.

7. Re-run your report and the data will match your themed style.

Removed uDig Functions still appear on Mac OSX

Some existing uDig functions continue to appear in the application and causes some confusion as no functionality is associated with these options.

To fix this is to run SMART with the -clean argument.

<code>SMART.exe -clean </code>
